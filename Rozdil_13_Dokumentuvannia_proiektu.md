РОЗДІЛ 13. ДОКУМЕНТУВАННЯ РЕЗУЛЬТАТІВ ПРОЄКТУ
13.1 Призначення документації
Документація є невід’ємною частиною проєкту та забезпечує:
розуміння архітектури системи;
можливість подальшої підтримки;
спрощення масштабування;
передачу проєкту іншим розробникам.

13.2 Основні документи проєкту
У рамках розробки інформаційної системи було створено:
Опис предметної області;
Аналіз вимог;
User Stories;
UML-діаграми;
Опис архітектури;
Модель даних;
Обґрунтування вибору БД;
Аналіз масштабування;
Аналіз безпеки;
Перспективи розвитку;
Вибір методології.

13.3 Технічна документація
Технічна документація повинна містити:
опис структури бази даних;
опис API (за наявності);
інструкцію з розгортання системи;
вимоги до середовища;
інструкцію для адміністратора.

13.4 Користувацька документація
Передбачено створення:
інструкції для адміністратора;
інструкції для страхового агента;
інструкції для менеджера;
інструкції для клієнта.

13.5 Висновок
Належне документування забезпечує підтримку, розвиток та масштабування системи у майбутньому та підвищує якість програмного продукту.
